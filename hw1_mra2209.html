Loading data

data("early_january_weather")

The data set contains hourly meteorological data for the airports LGA, JFK, and EWR for the month of January 2013. It contains the following variables:

  • origin, indicating the weather station.

  • year, month, day, hour, indicating the time of recording.

  • temp, dewp, indicating the temperature and dewpoint (in F).

  • humid, indicating the relative humidity.

  • wind_dir, wind_speed, wind_gust, indicating the wind direction (in degrees), speed and gust speed (in mph).

  • precip, indicating the precipitation (in inches).

  • pressure, indicating the sea level pressure (in millibars).

  • visib, indicating the visibility (in miles).

  • time_hour, indicating the date and hour of the recording.

The data set has 358 rows and 15 columns. The mean temperature is 39.58 degrees Fahrenheit.

Creating a Scatter Plot

weather_plot <- early_january_weather |> 
  ggplot( aes ( x = time_hour, y = temp, color = humid )) +
  geom_point(alpha = 0.7) + 
  labs(
      title = "Temperature over Time, Colored by Humidity",
      x = "Time (date & hour)",
      y = "Temperature (°F)",
      color = "Humidity"
  ) + 
  scale_x_datetime(
    date_breaks = "48 hours",
    date_labels = "%b %d\n%H:%M"
  ) +
  theme_minimal() + 
  scale_color_gradient(low = "blue", high = "red") + 
  theme(plot.title = element_text( 
    hjust = 0.5,
    face = "bold",
    size = 14
    )
  )

weather_plot

In the Temperature over Time plot above, we can see how the temperature around the LGA, JFK, and EWR airports changes over time during early January 2013. There appears to be a general day-night cycle in the temperatures. The humidity color gradient shows that higher humidity is usually associated with warmer temperatures and that low humidity is usually associated with colder times. There may also be periodic shifts due to weather fronts or the time of day. r

I am able to pull the mean of sample_nums and is_positive because the numbers in sample_nums are numeric and in is_positive the logical values are being treated as binary (1/0). However, the means of char_vec and factor_vec cannot be pulled because char_vec contains letters, which cannot be averaged, and factor_vec are being treated as cateogies, which can also not be averaged.
